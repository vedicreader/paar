The app

Inline CSS + hljs/CodeJar editor scripts, and the module state every route shares. The same app serves four frontends: the inline Jupyter iframe, the browser, paar-tui (JSON API), and agents (/agent/api).

Owner token & agent session

Two privilege tiers, enforced server-side. Owner endpoints that mutate the real namespace (/exec, /set, /api/exec) are gated by a random per-server token, written to <reg_dir>/token-<port> (0600): the server-rendered UI embeds it in its htmx headers and paar-tui reads the file, while agents are never handed it. Agents get a lazily-created AgentSession (see agent) — reads fall through to the owner namespace, writes stay in the agent’s layer, every cell passes the restriction list. agent_layer() is the owner’s window into what the agent has built; token gating is a courtesy lock against well-behaved local tools, not a cryptographic boundary (see the security notes in the index).

Agent surface: /agent/api

What paar-mcp (and any agent-side client) talks to. Same row/grid shapes as /api/*, but everything flows through the AgentSession overlay + policy, and exec scope obeys the server’s agent= mode: restricted allows the persistent overlay, readonly forces isolated, off 403s the whole surface. No owner token is ever required here — restriction, not authentication, is the control.

Live refresh & server lifecycle

def serve(
    port:int=8000, ns:NoneType=None, name:NoneType=None, poll:float=0.5, profile:str='standard',
    agent:str='restricted', token:bool=True, agent_kernel:bool=True
):

Start the paar inspector in a background thread for a plain (non-Jupyter) process; returns its base URL.

ns defaults to the caller’s module globals; name defaults to the repo/package/cwd label. Refresh is driven by a poll+diff thread, so any other environment can point paar-tui (or a browser) at the returned URL. agent: ‘restricted’ (default) serves the policy-guarded /agent/api overlay, ‘readonly’ forces isolated agent exec, ‘off’ disables the agent surface. token=True mints an owner token (written to /token-) gating /exec, /set and /api/exec.
